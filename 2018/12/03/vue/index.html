兼容性：>=ie9
vue优势：

  • 数据绑定非常简单，方便，灵活，核心只关注视图层
  • 轻量级，20k
  • 适用于移动端，适合个人或小型团队开发（同样适合大型项目，具体情况具体分析）
  • 中文文档，入门快
  • 渐进式框架（有选择的获取所需），非常方便的加载对应插件

script标签引入

npm安装

在npm包dist/目录下有多个vue构建版本，差别如下

差别 UMD——可以用于<script>引入 CommonJS——配合老的打包工具，Browserify和webpack 1 ES Module——配合现代打包工具，webpack 2和Rollup
完整版——包含编译器和运行时版本 vue.js vue.common.js vue.esm.js
只包含运行时版——比完整版小30%左右 vue.runtime.js vue.runtime.common.js vue.runtime.esm.js
完整版（生产环境） vue.min.js - -
只包含运行时版（生产环境） vue.runtime.min.js - —
  • 编译器：用于将模板字符串编译成为JavaScript渲染函数的代码（当传入一个字符串给template选项时需要）
  • 当使用vue-loader或vueify时，会在构建时预编译成JavaScript，此时，只想要运行时版本即可
  • 运行时：用于创建Vue实例，渲染并处理虚拟DOM等代码（基本是除去编译器的其他一切）

模板语法

数据绑定

{ {} }

{ {msg} }——Mustache语法（小胡子语法）

  • v-text的简写，同样不能识别html标签，最常用
  • 除了赋值，还能放置JS表达式如三元运算符（只能包含单个表达式），或者函数（必须加()），显示的是执行的返回结果。

放置函数，如{ {sum()} }，这种用法非常暴力，组件上任何数据变化都会重新执行sum函数，慎用

v-text

<div v-text="msg"></div>效果和<div>{ {msg} }</div>相同，不能识别html标签，默认不会作为属性显示在html上，当编译好后会自动插入元素中

v-html

和v-text区别：当绑定数据是html字符串是，能识别html标签

不能用于复合局部模板，因为vue不是基于字符串的模板引擎

v-once

无属性值，用于让数据只绑定一次时，当数据更改时，元素或组件及其所有子节点会被视为静态内容跳过，不会更新内容，如<div v-once>{ {msg} }</div>

能用于创建低开销的静态组件（优化更新性能），不能过度使用这个模式

v-model（双向数据绑定）

当改变数据时，输入框的值随之改变，相反输入框的值发生变化是，数据随之改变。<input v-model="msg">等价于<input v-bind:value="msg" v-on:input="msg=$event.target.value">

必须是oninput事件，不能是onchange事件

  • 修饰符
    • .lazy——取代<input>监听change事件
    • .number——输入字符串转为数字
    • .trim——输入首尾空格过滤

是表单元素<input>、<textarea>、<select>标签的行内属性，能实现双向数据绑定

v-cloak

无属性值，防止数据更新时闪烁，配合css样式用，可以隐藏未编译的小胡子标签直到实例准备完毕，css代码如下，属性选择器：

其他内置指令

类似AngularJS，指令是一种特殊自定义行间属性，vue中指令带有前缀v-

《数据绑定》章节中v-text、v-html、v-once、v-model和v-cloak都是vue的内置指令，其他内置指令如下。

v-show

  • 指令值为true或false，控制元素的显示或隐藏，效果等同设置displaycss属性
  • 不管值为什么，元素总会被渲染，只是基于css进行显示隐藏
  • 比v-if有更高的初始渲染开销但切换开销更低，如果切换频繁，使用v-show而不是v-if

v-show不支持<template>元素，并且不能和v-else指令共存

v-if、v-else-if、v-else（条件渲染）

  • 指令值为true或false，控制条件块内事件监听器和子组件的销毁和重建
  • v-if是惰性的，只有值为true时开始渲染块
  • 比v-show有更高的切换开销但初始渲染开销更低，如果值变化很少，使用v-if而不是v-show
  • v-else和v-else-if必须与v-if结合使用
  • 条件块内元素相同时，添加唯一值的key属性，以阻止元素的复用（就地复用策略是高效的，但建议尽可能提供key，除非刻意依赖默认行为以获取性能上的提升）
  • v-for比v-if有更高的优先级，如果想有条件的跳过循环的执行，请将v-if置于外层元素或<template>元素上
  • 在<template>元素上使用v-if，同时切换多个元素，而渲染结果不包含<template>元素

v-for（列表渲染）

  • 遍历对象，按Object.keys()的结果遍历，在不同浏览器下结果可能不同
  • 在更新v-for渲染的元素列表时，默认用“就地复用”策略，建议为每项提供唯一key属性，除非遍历输出的DOM内容非常简单，或者刻意依赖默认行为以获取性能上的提升
  • v-for比v-if有更高的优先级
  • 在<template>元素上使用v-for，同时渲染多个元素，而渲染结果不包含<template>元素
  • 对组件使用v-for时，key属性是必须的。并且必须用属性把数据传递到组件里

v-pre

无属性值，用于跳过此元素或组件及其子节点的编译，可以用于在页面显示原始的{ {} }字符，跳过大量没有指令的节点能加快编译。

v-on（简写@）（事件处理）

  • 绑定事件，普通元素上只能监听原生DOM事件。自定义组件上可以监听子组件触发的自定义事件。

  • 值：

    • 方法名，v-on:keydown="handle"
    • 一个内联语句，内联语句中事件对象为$event，@click="handle("ok",$event)"
    • 不带参数时绑定事件监听对象，不支持修饰符。v-on="{mousedown:handleMousedown,mouseup:handleMouseup}"

  • 修饰符

    • .stop——调用e.stopPropagation()，阻止冒泡传播。
    • .self——当e.target为事件绑定的元素本身时才触发
    • .prevent——调用e.preventDefault() ，阻止默认行为。
    • .capture——捕获模式，设置事件在捕获阶段中监听（默认在冒泡阶段中监听）
    • .{keyCode|keyAlias}——键盘特定键触发
      • vue为常用按键提供了别名（.enter、.tab、.delete——删除和退格键、.esc、.space、.up、.down、.left、.right，.f1）
      • 系统修饰键——按下相应键时才有可能触发鼠标或键盘事件（keyup.ctrl只释放ctrl按键不会触发事件，如果需要，改用keyCode，keyup.17）（.ctrl、.alt、.shift、.meta——win键）
      • .exact控制由精确的系统修饰符组合触发的事件，@click.ctrl.exact有且只有当ctrl按下时触发事件，@click.exact没有任何系统修饰键按下时执行
    • .native——监听组件根元素的原生事件
    • .once——只触发一次
    • .left——鼠标左键触发
    • .right——鼠标右键触发
    • .middle——鼠标中键触发
    • .passive——以{passive:true}模式添加侦听器（滚动事件的默认行为会立即触发，尤其能提高移动端性能）
  • @click.prevent.self阻止所有点击，@click.self.prevent只阻止元素自身的点击
  • 除了.once能被用于自定义组件事件，其他修饰符只能用于原生DOM事件
  • .passive能提升移动端性能，.prevent和.passive一起使用，.prevent会被忽略

v-bind（简写:）

  • 属性绑定值不为字符串类型时，即使是静态的，也需要用v-bind动态绑定，以表示值是一个JS表达式而不是字符串
  • 修饰符
    • .camel——将 kebab-case 特性名转换为 camelCase（驼峰命名），只能用于普通HTML特性的绑定. (从 2.1.0 开始支持)
    • .sync—— 语法糖，会扩展成一个更新父组件绑定值的 v-on 侦听器（双向绑定，只能用于prop绑定）。
    • .prop——用于绑定DOM属性（property）

      property和attributes差别：以<input id="example" type="text" value="name">为例，getDocumentById("example").value返回"Jack"，而getDocumentById("example").getAttribute("value")返回"name"

Class动态绑定

  • 绑定的表达式结果除了字符串，还可以是对象、数组。
  • 动态class能与普通class共存
  • 自定义组件上使用class时，会添加到该组件根元素上

Style动态绑定

  • 可以绑定对象，数组（数组可以将多个样式对象应用到一个元素上）
  • CSS属性名可以用驼峰式或短横线分隔（background-color，但要用单引号括起来）
  • vue会自动侦测并为需要的css属性添加浏览器引擎前缀
  • 2.3.0版本开始，css属性值能是一个包含多个带有前缀值的数组

特殊特性

key

用于Vue虚拟DOM算法，在新旧节点对比中，不使用key，Vue使用的算法会最大限度减少动态元素并修复再利用相同类型元素；而使用key，Vue算法会基于key变化重新排列元素顺序，并会移除key不存在的元素。

  • 值类型：number或string
  • 用于强制替换元素或组件而不是重复使用它。用处如下：
    • 完整地触发组件生命周期钩子
    • 触发过渡

有相同父元素的子元素必须有独特的key，重复的key会造成渲染错误

ref

用于给元素或子组件注册引用信息，注册在父组件的$refs对象上

  • 普通DOM元素上使用，指向真实的DOM元素
  • 子组件上使用，指向子组件实例
  • 带有v-for的元素或子组件，指向真实的DOM元素或组件实例组成的数组

ref作为渲染结果被创建，初始渲染时不存在，$refs不是响应式的

slot（插槽）

不使用<slot>向组件中传递内容时，这些子元素被存储在vm.$slots.default中

vue实例选项

  • new Vue({})实例就是ViewModel，它向vue的响应式系统中加入data对象中的所有属性，并代理data对象（vm.obj==obj =>true）

  • Vue类的属性与方法都有前缀$，与用户自定义属性区分

不要在选项属性或回调上使用箭头函数，因为this指向需要是当前实例。

el

提供页面上已存在的DOM元素作为Vue实例的挂载目标，可以是CSS选择器或html元素实例

  • 挂载后，元素通过vm.$el访问
  • 挂载元素会被Vue生成的DOM替换
  • 如果render函数和template函数都不存在，挂载DOM元素的HTML会被提取出来用作模板，必须使用Runtime+Compiler构建的Vue库

template

  • 字符串模板，会替换挂载元素，除非模板内容有分发插槽
  • 值为字符串类型模板；或以#开头的选择符，用<script type="text/x-template" id="my-template">包含模板

如果vue选项中存在render函数，字符串模板将被忽略

render（渲染函数）

如果render函数存在，vue构造函数不会从template选项或el选项元素中提取HTML模板编译渲染函数

renderError

render函数出错时，其错误作为第二个参数传递到renderError函数

  • 只在开发者环境下工作，配合热加载功能非常实用

data

  • 值为对象或返回值为对象的function
  • 当实例创建时，vue会递归地将data的属性转换为getter/setter，从而让data中存在的属性是响应式的（向vue的响应式系统中加入data对象中的所有属性）。而此过程仅在实例创建时发生，所以所有用到的 数据都需要在创建实例时初始化
  • data中_和$开头的属性不会被vue实例代理，只能通过vm.$data来访问
  • 使用Object.freeze()，会组织属性修改，响应系统不再追踪变化
  • 组件定义时，data的值只能是function类型，如果是一个纯粹的对象，所有实例将维护同一个数据对象

props

  • 值为数组或对象，用于接受来自父组件的数据，组件不能直接更新自己的属性

props的验证都在组件创建之前进行，所以验证中不能使用data，computed等属性

非props特性

指传入组件但是组件中没有定义属性

  • 会自动添加到组件根元素上
  • 绝大多数属性会替换组件根元素原有属性值，而class和style会合并原有和非props特征值

propsData

只用于new创建的实例中，创建实例时传递props，主要用于方便测试

computed（计算属性）

  • 以绑定数据的方式绑定计算属性，如{ {sum} }，而不能是{ {sum()} }。声明计算属性时，需要以函数形式（只需要get函数时）或者get和set函数组成的对象形式。
  • 计算属性具有缓存，当数据变化，计算属性才会重新计算，当依赖是非响应式属性，计算属性不会被更新。
  • 当获取计算属性值时，计算属性的get函数返回值赋予计算属性。当设置计算属性值时，set函数用于影响其他需要随之改变的值。
  • 计算属性如果使用箭头函数，this不会指向组件实例，但可以将实例作为函数第一个参数来访问
  • 计算属性与方法（methods）的区别：计算属性基于自己的依赖进行缓存，只有当依赖发生变化是才会重新执行；方法则只要触发渲染，就会再次执行。
  • 计算属性与侦听属性（watch）的区别：前者侧重结果（值），后者侧重影响（过程）。
    • 侦听属性也能响应数据变动。当一个值依赖多个数据时（多对一），为了更具可读性（代码的精简），使用计算属性更好；当一个数据的变化引起多个操作或数据变化时（一对多），侦听属性更好。
    • 计算属性返回state处理后结果，侦听属性是赋值腥味，修改state。当执行异步操作不能串行返回结果时；当开销较大操作，避免堵塞主线程时；使用侦听属性。当简单且串行返回时，使用计算属性。

watch（侦听器）

用于响应数据或计算属性变化，在数据或计算属性变化时，执行操作，执行异步或开销较大的操作尤其有用。

  • 因为this的指向问题，不要用箭头函数来定义侦听函数
  • 键：侦听的表达式。
  • 值：
    • 对应的回调函数
    • 方法名
    • 多个回调函数组成的数组
    • 包含选项的对象

回调函数得到的参数为新值和旧值，当变异值为对象或数组时，新旧值完全相同，因为新旧值指向同一空间地址，vue不会保留变异前的数据

methods

  • 可以直接通过实例访问方法，或直接在指令表达式中调用
  • 因为this的指向问题，不要用箭头函数来定义方法

生命周期

  • 所有生命周期钩子自动绑定this上下文到实例中，所以不能使用箭头函数来定义一个生命周期方法
  • beforeMount和mounted，beforeUpdate和updated，activated和deactivated，beforeDestroy和destroyed在服务器端渲染期间不被调用。

directives（自定义指令）

自定义钩子函数参数

  • el——指令绑定元素，可以直接DOM操作
  • binding——一个对象，包含属性如下
    • name——指令名（不包括v-前缀）
    • value——指令绑定值（绑定的是表达式时，值是表达式的结果），可以为对象数据类型
    • oldValue——绑定的上一个值（仅在update和componentUpdated钩子中可用）
    • expression——字符串形式的指令表达式
    • arg——传给指令的参数，v-demo:foo.a.b="mag"，arg为"foo"
    • modifiers——包含修饰符的对象，如{a:true,b:true}
  • vnode——Vue编译成的虚拟节点
  • oldVnode——上一个虚拟节点（仅在update和componentUpdated钩子中可用）

当bind和update触发相同行为，而不需要其它钩子时，指令定义值可以直接为函数数据类型

filters（过滤器）

对象数据的属性增删

数组

  • push()、pop()、shift()、unshift()、splice()、sort()、reverse()在Vue中是一组观察数组的变异方法，会触发视图更新。
  • vue还增加了两个方法set()、remove()
  • 其余方式（如vm.ary[index]=newValue）更改数组不会触发视图更新，这种情况可以用Vue.set(vm.ary,index,newValue)或vm.$set(vm.ary,index,newValue)实现对数组的更改

对象

  • Vue不能检测对象属性的添加或删除，不能动态添加根级别的响应式属性。但可以使用Vue.set(vm.obj,key,value)或vm.$set(vm.obj,key,value)方法向嵌套对象添加响应式属性

规范及要点

组件名

  • 每个组件分成单独文件，文件名和组件名始终完整单词开头字母大写
  • 组件名始终是多个单词，除了根组件（为了避免与任何时候的HTML元素冲突）
  • 组件名以高级别单词开头，描述性的修饰词结尾
  • 组件名使用完整单词而不是缩写，因为明确性非常宝贵

  • 展示类组件（无逻辑、无状态组件）都以一个特定前缀开头，如Base，好处如下：

    • 编辑器中基础组件会列在一起，容易识别
    • 基础组件被频繁使用，可以放到全局，而不用分别导入它们。

  • 单例组件（每个页面只使用一次组件，不接受任何prop），以The前缀命名，以示其唯一性

  • 和父组件紧密耦合的子组件以父组件名作为前缀，编辑器可以将关联文件排在一起

不建议过多嵌套子目录，因为会增加编辑器侧边栏浏览组件所花时间。除非非常大型的应用（100+组件）

模板注意事项

  • v-for设置键值key（除非少数对性能有严格要求的情况，避免对象固化）
  • v-for和v-if不要作用在同一元素上，如果只需要显示某些项，通过过滤数据的方式实现
  • DOM模板中不要使用自闭合组件，因为HTML不支持自闭合的自定义元素。单文件组件、字符串模板和JSX中没有内容的组件应该自闭合，如<MyComponent/>
  • DOM模板中使用自定义组件，组件名不能使用首字母大写+驼峰命名法，因为HTML对大小写不敏感，要使用为-分隔单词命名
  • 模板及JSX中属性名要使用为-分隔单词命名
  • 元素多个属性应分行写，各占一行
  • 元素属性应该始终带引号（选用JS中不用的单引号或双引号）
  • 指令缩写全局统一用或者不用

选项注意事项

  • 组件data必须是一个函数
  • props中的属性定义尽可能详细，至少指定类型
  • 属性命名使用驼峰命名法
  • 自定义私有属性名使用$_前缀，并附带一个命名空间，以避免冲突，如$_yourPluginName_
  • 把复杂的计算属性分割成可能多的更简单的多个计算属性，使代码更易于测试、易于阅读、更好的应对需求变化
